Known issues / risks

Things that work but could break, or quirks we know about.

LinkedIn DOM is moving target
Risk
Selectors break when LinkedIn rebuilds components. We mitigate with multi-fallback selector chains (8 levels deep in dom-reader.ts), but a full LinkedIn React rewrite would force a rescue mission. Watch for: messaging DOM (Phase 1 in progress), search results, profile page.
LinkedIn rate-limit detection
Risk
We rate-limit ourselves (25 profiles/session, 2 sessions/day) but if LinkedIn shadow-throttles or shows CAPTCHA we currently don't detect or surface that gracefully. Hit-the-wall UX needs design.
Single-user dev mode
Risk
Auth via Supabase is wired but multi-user flows haven't been battle-tested. RLS policies exist but need an audit before any real user invites.

Key decisions (do not override)

Chat-first interface
Primary interaction is conversation with an AI agent, not a dashboard. Contacts and Goals are outcome views displaying results of conversations.
Session-per-contact architecture
Each contact gets a dedicated chat thread. General Chat for strategy/discovery; Contact Sessions for working on specific people. Artifacts produced in a session attach to that contact's permanent profile card.
Browser companion, not API
Chrome Extension Manifest V3 running in user's own session. No third-party data APIs. Proxycurl was sued; Apollo and Seamless were banned. Browser companion is the only safe architecture (hiQ Labs precedent).
MiniMax for LLM (replaces Claude)
FAST tier for scoring/simple, REASONING for complex. Structured JSON outputs everywhere. Rolling summarization for long conversations.
profile_md as the user's identity narrative
Markdown stored on users.profile_md. Built once from onboarding, enriched over time. Injected into every prompt so the coach knows the user. Will be surfaced as editable artifact in Settings view (next).
Diagnostic-first when iteration fails
When a fix iteration fails, INSTRUMENT before guessing. The Bain saga (6 failed attempts) was finally solved by adding [CDP DIAG] logs — the next test surfaced the exact 1-line bug. Pattern adopted for all future debug sessions, including current messaging Phase 1.
