## Agent B, Meeting Prep `/v2/contacts/[id]/prep`

Port meeting-prep.jsx. Launched from Contact Detail's "Prep meeting" action (Agent B also adds that button/link in the detail sidebar, coordinate: the link target is `/v2/contacts/{id}/prep`; if editing the detail sidebar risks a conflict, instead the orchestrator will add the link, DO NOT edit `src/components/v2/contacts/**`; just build the prep route and tell the orchestrator the link target).

Flow (REAL MiniMax):
1. `GET /api/contacts/{id}` for header/context. Loading.
2. Intake form (purpose, duration, goal, focus), local state. On submit:
3. `POST /api/conversations { type:"contact_session", contact_id }` → conversation_id.
4. `POST /api/ai/generate { artifact_type:"meeting_prep", contact_id, conversation_id, user_instructions: <intake summarized> }` → `{data:{ artifact_id, content }}`. **The `content` shape is:**
   ```
   { person_summary: string,
     company_intel: { description: string, recent_news: [{headline, date?}], strategic_priorities: string[] },
     discussion_themes: [{ name: string, questions: string[] }],
     coaching: { do_list: string[], dont_list: string[], positioning_advice: string, recommended_ask: "advice"|"referral"|"introduction"|"none yet" } }
   ```
   (Confirm by reading `src/lib/ai/generation.ts` ~line 213. Render defensively, any field may be missing.)
5. Render the brief like the design's meeting-prep: a Snapshot (person_summary + company_intel.description + coaching do/don't), Company (recent_news + strategic_priorities), a phased Question bank from `discussion_themes` (theme name + its questions, each with room for the "why"/notes), and the Ask (coaching.recommended_ask + positioning_advice).
   - The design also has an **Agenda** (timed blocks) and **live notes** the backend doesn't return. Agenda = a sensible local default template derived from the chosen duration (clearly a template). Live notes = local textarea; "Save notes" appends to `contact.notes` via `PUT /api/contacts/{id} { notes }` (reuse that contract; don't invent an endpoint).
6. Loading (shimmer/skeleton while generating, meeting_prep is a heavier call, may take 5-15s; show clear progress), error+retry states.
7. Components in `src/components/v2/prep/**`. Do NOT edit primitives.tsx or `src/components/v2/contacts/**`.
